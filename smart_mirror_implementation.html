4.1 Файловая структура проекта

Каждый плагин — отдельный Python-пакет. Контур выбора устройств и централизованный UI вынесены в отдельные сервисы и интерфейсы.

smart_mirror/ ├── main.py # Точка входа · создаёт EventBus, PluginManager, DeviceManager ├── config.yaml # Какие плагины включены + параметры ├── core/ │ ├── event_bus.py # EventBus — асинхронная шина событий │ ├── plugin_manager.py # PluginManager — загрузка и управление │ ├── device_manager.py # DeviceManager — discovery, saved selection, validate │ └── config.py # Config — чтение config.yaml ├── interfaces/ │ ├── plugin.py # IPlugin — базовый (start, stop, name) │ ├── sensor.py # ISensorPlugin → IPlugin (capture) │ ├── processor.py # IProcessorPlugin → IPlugin (process) │ ├── device.py # IDeviceProvider / IDeviceRegistry │ ├── ui.py # IUIContract — мастер устройств + стандартные UI-события │ ├── output.py # IOutputPlugin → IPlugin (render) │ └── storage.py # IStoragePlugin → IPlugin (save, load) ├── models/ │ ├── events.py # Event — структура событий EventBus │ ├── device.py # DeviceInfo, SelectedDevices, DeviceValidation │ └── types.py # VideoFrame, AudioChunk, TestResult, UICommand, ... ├── plugins/ │ ├── camera/ # CameraPlugin → ISensorPlugin │ ├── microphone/ # MicrophonePlugin → ISensorPlugin │ ├── video_analysis/ # VideoAnalysisPlugin → IProcessorPlugin │ ├── voice_test/ # VoiceTestPlugin → IProcessorPlugin │ ├── voice_interaction/ # VoiceInteraction → IProcessorPlugin (→ ИИ) │ ├── aggregator/ # AggregatorPlugin → IProcessorPlugin [ЯДРО] │ ├── ai_assistant/ # AIAssistantPlugin → IProcessorPlugin [ОТКЛ.] │ ├── ui/ # UIPlugin → IOutputPlugin + централизованные экраны/адаптеры │ ├── report/ # ReportPlugin → IOutputPlugin │ ├── storage/ # StoragePlugin → IStoragePlugin │ └── training/ # TrainingPlugin → IProcessorPlugin └── logs/ ├── sessions/ # Данные сеансов (.json) ├── reports/ # Отчёты (.json, .csv) ├── events/ # Лог событий EventBus └── training/ # Результаты тренировок

4.2 Иерархия интерфейсов

Все плагины наследуют от IPlugin. Интерфейсы устройств и UI-контракты вынесены отдельно, чтобы менять экраны и выбор устройств без правок в обработчиках.

ИнтерфейсНаследуетРасположениеДля чегоКто реализует
IPluginABCinterfaces/plugin.pyБазовый: старт, стоп, имя, версияВсе плагины
ISensorPluginIPlugininterfaces/sensor.pyДатчики: непрерывный захват данныхCamera, Microphone
IProcessorPluginIPlugininterfaces/processor.pyОбработка: получает событие → возвращает результатVideoAnalysis, VoiceTest, VoiceInteraction, Aggregator, AIAssistant, Training
IDeviceProviderABCinterfaces/device.pyDiscovery, сохранение и проверка выбора устройствDeviceManager
IUIContractABCinterfaces/ui.pyМастер выбора устройств, стандартные UI-экраны и UI-событияUIPlugin
IOutputPluginIPlugininterfaces/output.pyВывод: рендер экрана, генерация отчётаUI, Report
IStoragePluginIPlugininterfaces/storage.pyХранение: async запись/чтение файловStorage

4.3 Плагины и сервисы — внутренний процесс каждого

Пошаговое описание работы каждого компонента. Что происходит внутри от получения события до публикации результата.

КомпонентИнтерфейсВХОД (подписка)ВЫХОД (публикация)Внутренний процесс (пошагово)
DeviceManagerIDeviceProvider ui.device_selected cmd.prepare_session device.selection_resolved device.validation_failed ui.device_wizard.open 1. Читает сохранённый выбор → 2. Перечисляет доступные камеры и микрофоны → 3. Сравнивает сохранённые device_id с фактически доступными → 4. Открывает UI-мастер выбора устройств → 5. При подтверждении сохраняет новый выбор → 6. Публикует SelectedDevices для старта сеанса или ошибку/статус, если устройство недоступно.
CameraPluginISensor cmd.start_capture sensor.video_frame 1. Получает команду старт и уже разрешённый selected_camera_id → 2. Открывает конкретную камеру → 3. В цикле захватывает кадры (≥25fps) → 4. Каждый кадр упаковывает в VideoFrame → 5. Публикует в EventBus → 6. Не выполняет discovery или выбор устройства самостоятельно.
MicrophonePluginISensor cmd.start_capture sensor.audio_chunk 1. Получает команду старт и уже разрешённый selected_microphone_id → 2. Открывает конкретный микрофон (16kHz) → 3. В цикле записывает аудиочанки → 4. Упаковывает в AudioChunk → 5. Публикует в EventBus. Логика discovery/select здесь отсутствует.
VideoAnalysisPluginIProcessor sensor.video_frame analysis.video_result 1. Получает кадр → 2. Детекция лица → 3. Извлечение ключевых точек → 4. Классификация эмоций → 5. Анализ направления взгляда → 6. Оценка двигательной активности → 7. Упаковка в ScreeningResult → 8. Публикация
VoiceTestPluginIProcessor sensor.audio_chunk cmd.start_test test.voice_result 1. Ждёт команду запуска теста → 2. Начинает сбор аудио → 3. Распознаёт речь (ASR) → 4. Анализирует корректность ответов → 5. Считает баллы и время реакции → 6. Классификация (норма/отклонение) → 7. Публикация TestResult
VoiceInteractionPluginIProcessor sensor.audio_chunk ai.tts_text voice.intent voice.tts_output 1. Получает аудио → 2. Распознаёт речь (ASR) → 3. Извлекает намерение (NLU) → 4. Публикует Intent для ИИ-ассистента. По запросу TTS: 5. Получает текст → 6. Синтезирует речь → 7. Воспроизводит. Привязан к ИИ-ассистенту.
AggregatorPluginIProcessor analysis.* test.* ui.action ai.command device.selection_resolved cmd.* report.data storage.write Ядро системы. 1. Инициирует подготовку сеанса → 2. Ждёт device.selection_resolved и только затем разрешает старт сенсоров → 3. Собирает результаты от обработчиков → 4. Мультимодальное слияние (видео + аудио) → 5. Оценка когнитивного состояния → 6. State Machine переходит в следующее состояние → 7. Отправляет команды: запуск камеры/теста/UI → 8. Передаёт данные в отчётность → 9. Пишет логи. Состояния: IDLE → READY → SCREENING → VOICE_TEST → REPORTING → TRAINING → IDLE
AIAssistantPlugin откл.IProcessor voice.intent external.command ai.command ai.tts_text 1. Получает намерение от VoiceInteraction → 2. Или получает внешнюю команду (API) → 3. Формирует команду для Агрегатора → 4. Отправляет текст для озвучки → TTS. Если отключён — система работает через кнопки UI, голосовые команды недоступны.
UIPluginIOutput cmd.ui_update report.summary training.suggest ui.device_wizard.open device.validation_failed ui.action ui.device_selected 1. Получает команду от Агрегатора или DeviceManager → 2. Рендерит экран из централизованного набора экранов, форм, кнопок и адаптеров → 3. Показывает мастер выбора устройств, тесты, результаты и экран занятий → 4. Пользователь выбирает устройство или нажимает кнопку → 5. Публикует стандартизованные UI-события. Все изменения экранов и UI-потоков локализованы в этом модуле.
ReportPluginIOutput report.data report.summary storage.write 1. Получает данные скрининга → 2. Формирует полный отчёт (аналитика, тренды) → 3. Краткую версию отправляет на UI → 4. Полную версию отправляет в хранилище (JSON/CSV)
StoragePluginIStorage storage.write storage.read storage.read_result 1. Получает данные для записи → 2. Async записывает в /logs/ (JSON/CSV) → 3. По запросу Агрегатора читает историю → 4. Возвращает исторические данные. Полностью async I/O — не блокирует event loop.
TrainingPluginIProcessor report.data training.suggest storage.write 1. Получает данные скрининга → 2. Анализирует слабые стороны → 3. Подбирает когнитивные упражнения → 4. Адаптирует сложность → 5. Отправляет предложение занятий на UI → 6. Сохраняет результаты тренировок
