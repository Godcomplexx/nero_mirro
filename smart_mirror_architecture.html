Архитектура программного обеспечения системы «Умное зеркало»

Модульная интеллектуальная система мониторинга когнитивного и функционального состояния

Таблица 1. Описание модулей системы и потоков данных

Структурированное описание всех компонентов архитектуры. Перед стартом сеанса система проходит контур инициализации устройств: Device Manager обнаруживает камеры и микрофоны, проверяет сохранённый выбор и открывает централизованный UI-мастер выбора устройств. Модуль голосового взаимодействия (2.1) по-прежнему привязан к ИИ-ассистенту.

Таблица 3. Замкнутые циклы обработки данных

Циклы обратной связи. Перед началом тестирования система проходит обязательный контур подготовки сеанса: выбор и проверку устройств. ИИ-ассистент участвует опционально — система работает и без него.

Цикл Последовательность Тип данных Латентность Назначение
Цикл инициализации Пользователь → Интерфейс (мастер) → Device Manager → Камера / Микрофон
Device Manager → Агрегатор 		Список устройств → выбор → selected_device_id → статус готовности 1–3 с Предсеансовая подготовка. Система перечисляет доступные устройства, подставляет сохранённый выбор, валидирует его и не запускает сенсоры, пока выбор не подтверждён.
Цикл взаимодействия Пользователь → Интерфейс (кнопки) → Агрегатор → Интерфейс (вывод) → Пользователь Действия пользователя → команды → визуальный вывод < 500 мс Основной цикл реального времени после успешной инициализации устройств. Агрегатор самостоятельно управляет интерфейсом. ИИ-ассистент не обязателен.
Сеанс скрининга Интерфейс / ИИ (опц.) → Агрегатор → Камера / Микрофон → VideoAnalysis / VoiceTest → Агрегатор → Отчётность / Занятия → Интерфейс UI-события → команды сценария → видео/аудио → результаты обработки → отчёт/занятия 5–15 мин Основной контур сеанса после успешной инициализации. Агрегатор запускает и координирует обработку, получает результаты модулей, а затем передаёт данные в отчётность и подбор занятий.
Цикл отчётности Агрегатор → Модуль отчётности → Папка логов (/logs/)
Модуль отчётности → Интерфейс зеркала → Пользователь 		Агрегированные результаты → отчёт (.json/.csv) + визуализации 2–5 с Формирование отчётов и сохранение в файловое хранилище. Пользователь видит краткие результаты на экране зеркала.
Цикл адаптации Папка логов → Агрегатор → Изменение сценария → Новый тест / режим / занятия
↓ результаты → /logs/ (замыкание) 		Исторические файлы → параметры адаптации → обновлённый сценарий Межсеансовый Перед сеансом Агрегатор читает логи и адаптирует тесты, сложность и набор занятий. Результаты тренировок также влияют на следующий сеанс.
